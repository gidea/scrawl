Scrawl is [open source](https://github.com/gidea/scrawl) and we strive to be fully transparent about what we're working on and what's next. This roadmap is a living document updated as we make progress.

## Inherited from Emdash

Scrawl inherits a mature feature set from [Emdash](https://github.com/generalaction/emdash), the Agentic Development Environment we forked from. These features work today:

- Parallel agent orchestration with 22+ CLI agents
- Git worktree isolation for each task
- Diff view and inline editing
- Kanban board for task management
- Issue integration (Linear, Jira, GitHub Issues)
- MCP server management
- Skills system (Agent Skills standard)
- SSH remote development
- CI/CD check monitoring
- tmux session persistence

## Writer-Specific Features (Planned)

These are the features we're building specifically for writers:

- **Cloud vector database integration** — Connect to vector databases via MCP for retrieval-augmented generation (RAG), enabling research-backed writing workflows where agents cite sources rather than hallucinate them
- **Writer-specific skills** — Reusable prompt packages for style guide enforcement, tone matching, audience adaptation, and format compliance (e.g., "apply AP style", "match our brand voice", "format as a technical spec")
- **Content project templates** — Pre-configured project setups for common writing workflows: blog posts, proposals, technical documentation, marketing copy
- **Writer-optimized UI** — Interface refinements tailored to content creation workflows, reducing friction for non-developer users
- **Source attribution in RAG workflows** — Agents surface the sources they drew from, reducing silent fabrication

## Our Stance on AI-Generated Content Quality

We take the risk of AI slop seriously. Agentic writing can produce generic, homogenized content that erodes distinct voice and degrades the quality of what gets published. Our answer is not to avoid agentic writing — it's to build better guardrails into the workflow:

- **Best-of-N comparison**: run multiple agents on the same brief and compare drafts before choosing one. Different agents produce meaningfully different output.
- **Structured diff review**: every change an agent makes is visible and human-reviewable before it merges. Nothing lands silently.
- **Style enforcement via skills**: install a skill that encodes your house style, tone guidelines, or audience profile. Agents apply it on every run, reducing drift toward generic defaults.
- **Source attribution**: planned for RAG workflows — agents cite what they drew from, so fabricated claims are easier to catch.

The goal is augmented writers, not replaced ones. The writer sets the brief, reviews the output, and makes the final call.
